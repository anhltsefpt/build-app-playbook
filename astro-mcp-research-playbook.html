Astro MCP Research Playbook

App Idea Validation System for Anh @ LineCentury — Created 2026-05-14

Purpose: A reusable system for validating mobile app ideas using Astro MCP (App Store keyword research tool). Designed for a solo indie dev who copies + improves proven markets, not reinvents them.

Philosophy: Kill fast, dive deep on survivors. Domain experience + monetization potential > market size.

1. Problem Statement

As a solo indie developer with 107 validated app ideas but only 5-6 active products, the biggest bottleneck is not ideation — it's decision paralysis. Too many ideas survive too long without a kill signal, spreading execution thin across too many fronts.

Manual App Store research is slow (30-60 min per idea). Gut feel isn't reliable enough to kill ideas you're emotionally attached to. Data-backed scoring is needed to disqualify fast and commit with confidence.

2. Solution

A two-phase validation pipeline powered by Astro MCP:

1
Phase 1: Kill Filter — Fast disqualification. Run Astro queries on all ideas. Score each. Kill anything below threshold. Goal: eliminate 60-80% of ideas in one session.
2
Phase 2: Deep-Dive — Survivors get full competitive intel. Go/no-go scorecard per idea. Output: ranked table with top 3 breakdowns. Goal: commit to 1-2 ideas.

3. Scoring System

Weighted Scorecard

Dimension Weight Source How to Score (0-100)
Monetization Signal 3x Astro: competitor pricing 0 = all competitors free, no IAP
30 = some have IAP but low prices ($0.99-1.99)
60 = multiple competitors charge $4.99+/yr subscription
80 = competitors earn $50K+/yr (visible in rankings)
100 = proven $1M+/yr market (I Am Sober, Strong, etc.)
Idea Complexity 3x Your judgment 0 = calculator / single-screen utility (KILL)
30 = simple tracker with 2-3 screens
60 = multi-feature app with retention loop
80 = app with data, history, insights, progression
100 = deep product with personalization, AI, community
Keyword Depth 2x Astro: keyword_suggestions 0 = 0-2 related keywords with Pop > 10
30 = 3-5 keywords
60 = 6-12 keywords
80 = 13-25 keywords
100 = 25+ keywords with Pop > 10
Market Demand 1x Astro: primary keyword Pop Direct Pop score from Astro (0-100)
Competition Difficulty 1x Astro: keyword Diff (inverted) Score = 100 - Diff. High Diff = hard to rank = low score.
Domain Experience 1x Your judgment 0 = no experience, can't empathize (BINARY KILL)
50 = understand the category, not the specific pain
100 = lived this problem, feel the struggle daily

Score Formula

Total = (Monetization x 3) + (Complexity x 3) + (Keyword Depth x 2) + Market Demand + Competition + Domain Experience
Max possible = 300 + 300 + 200 + 100 + 100 + 100 = 1100

Normalized = (Total / 1100) x 100   → final score out of 100

Decision Thresholds

Score RangeVerdictAction
70-100Strong candidateDeep-dive immediately
50-69ConditionalDeep-dive only if < 3 strong candidates
0-49KillArchive, don't revisit for 6 months
Binary Kills (instant disqualification regardless of score):
• Domain Experience = 0 → KILL
• Idea Complexity = 0 → KILL (it's a calculator, not a product)
• Not a mobile app → KILL (current constraint)

4. Astro MCP Workflow

Phase 1: Kill Filter (30 min for ~12 ideas)

1
Pre-filter (YOU, 5 min)
Tag each idea: "I feel this pain" = yes/no. Kill all "no" immediately. Tag complexity: real product or calculator?
2
Astro: Market scan (per surviving idea)
Run live_app_store_search("[primary keyword]") for each idea.
Check: Do results exist? Are top apps charging money? How many results?
3
Astro: Keyword depth check
Run keyword_suggestions for each idea's primary keyword.
Count how many related keywords have Pop > 10.
4
Astro: Competitor monetization check
From search results, add top 3-5 competitors via add_apps.
Check their pricing model, ratings, review sentiment.
5
Score & rank
Fill scorecard per idea. Sort by total score. Kill everything below 50.

Phase 2: Deep-Dive (1-2 hours for top 3)

1
Astro: Full keyword map
Run keyword_suggestions + extract_competitor_keywords on top 5 competitors.
Build tier list: GOLD (Pop ≥ 30, Diff ≤ 35), SILVER, BRONZE.
2
Astro: Competitive landscape
get_ratings for top 5 competitors across US + target regions.
fetch_keywords to see what they rank for.
search_rankings to map their positions.
3
Gap analysis
Cross-reference competitor keywords with their ratings.
Identify: keywords they rank for but have bad reviews = your entry point.
4
Output: Go/No-Go scorecard
One-page summary per idea with final verdict and recommended ASO keywords for launch.

8. Kill Criteria

Hard Kill: No Domain Experience

"I don't have the struggle." Can't empathize with core user pain. No amount of market data fixes this.

Examples killed: PetScan (no pets), Halal Scanner (not Muslim), Ramadan Tracker (not Muslim)

Hard Kill: Too Simple to Monetize

"It's a calculator, not a product." Users won't pay because it's a Google search or a single formula. No retention loop, no depth.

Examples killed: UAE Gratuity Calculator, Gold Price Tracker, Tasbeeh Counter

Soft Kill: Low Astro Scores

Fails data validation even though domain + complexity are fine:

  • No competitors charge money (monetization signal = 0)
  • Keyword depth < 3 related keywords with Pop > 10
  • Primary keyword Pop < 5 with no adjacent opportunity
  • Total weighted score < 50

9. Implementation Decisions

Strategy: Copy + Improve

Not trying to build unique/novel apps. Find proven markets where competitors make money, copy the model, improve the UX. This means:

  • Monetization signal is the #1 weighted factor (3x) — if nobody pays, skip it
  • Competition difficulty is low weight (1x) — crowded is fine if they're beatable
  • "Beatable" = competitors rated 3.5-4.2 stars with common complaints you can solve
  • Don't need to be #1 — need to capture a profitable slice

Tech Stack (Fixed)

All ideas will be built with the same proven stack:

  • Mobile: React Native / Expo
  • Backend: Supabase (PostgreSQL + Edge Functions)
  • Monetization: RevenueCat (subscriptions + IAP)
  • Analytics: Amplitude (events + experiments + remote config)
  • ASO: Astro (keyword tracking + competitive intel)

Output Format: Ranked Markdown Table

After Astro analysis, results saved as a scored table in the brainstorming folder. One file per research session. Sortable by total score. Top 3 get expanded deep-dive sections below the table.

10. Constraints & Limitations

ConstraintImpactMitigation
iOS App Store only No Android market data Acceptable — iOS first, Android later. iOS users pay more.
60 req/min rate limit Can't bulk-analyze 100+ ideas in one session Pre-filter manually first. Batch operations. Phase work across sessions.
Requires Astro macOS app running Not a cloud API, needs local machine Run during dedicated research sessions, not ad-hoc.
Beta status Data may be inaccurate, API may break Cross-verify critical findings against App Store directly.
Mobile apps only (no games) Self-imposed scope constraint Focus enables faster execution.
Token consumption MCP calls cost Claude tokens Batch operations, cache results, don't repeat queries.

11. Out of Scope

  • Android market research — Astro is iOS/App Store only
  • Game ideas — user constraint, mobile apps only
  • Web/SaaS ideas — not in Astro's scope
  • Automated idea generation — user already has 107 ideas, problem is selection not creation
  • User interviews / qualitative research — known weakness but not addressed by this tool
  • Post-launch ASO optimization — covered by Astro but not in this validation playbook scope
  • Revenue projections — existing PRDs already have these; Astro validates market, not financial models
